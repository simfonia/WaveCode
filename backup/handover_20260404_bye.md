## 2026-03-08 ~ 2026-03-09 (略)

## 2026-03-10 (視覺化革命與診斷基石)
### 1. 本次對話達成
- **樂器定義帽子積木**：完全對齊 #processing 邏輯，實作「定義音色 -> 引用演奏」的模型。
- **SVG ADSR 視覺化**：開發了自定義 Blockly 欄位 `FieldADSR`，能在積木上繪製曲線並同步顯示演奏進度的「黃色跳動光點」。
- **即時示波器 (Oscilloscope)**：在主介面實作了磁滯觸發示波器，能靜止、穩定地觀察 Rust 輸出的原始波形。
- **音質診斷與優化**：
    - 解決了 Windows 多聲道環境下的緩衝區空洞雜音。
    - 實作了聲部物理隔離（Gate 物理斷路），消除了單音演奏時的並聯滲漏（拍音）。
    - 實作了頻率平滑 (`follow` 節點)，大幅改善了音符切換時的 Pop 聲。

### 2. 技術細節
- **磁滯觸發**：Rust 端的數據抓取採用了 `-0.05 ~ +0.05` 的磁滯區間，確保前端繪圖絕對靜止。
- **SVG vs Canvas**：棄用 Canvas 嵌入 SVG，改用 Blockly 原生 `createSvgElement`，解決了工作區縮放與 Minimap 顯示問題。

### 3. 下一步行動
- **核心穩定化**：目前正弦波在極快速連續演奏時仍有細微雜訊，需研究相位硬同步（Hard Sync）或在 Rust 端實作更細緻的 Crossfade。
- **功能擴充**：實作取樣器 (Sampler) 與 FFT 頻譜分析儀。

## 2026-04-04 (右側面板佈局與 UI 穩定性)

### 1. 視覺化佈局最佳化
- **示波器與頻譜並列**：修正了原本上下分割的問題，將 `visualizer-container` 的 `flex-direction` 改為 `row`，實現左右並列分割。
- **畫布高度固定**：為 `waveformCanvas` 與 `fftCanvas` 設定了固定的 `120px` 高度，確保在各種解析度下都能穩定顯示。

### 2. 面板收合與高度對齊
- **收合一致化**：解決了收合後示波器下方留有殘留高度的問題。透過將 `min-height` 限制移至 `:not(.collapsed)` 狀態，並在 `.collapsed` 下強制隱藏內容，確保兩面板收合後高度完全統一。
- **日誌捲軸修復**：修正了因 `height: auto` 導致日誌無限增長且捲軸消失的問題。現在展開狀態下固定為 `180px`，正確觸發內部捲軸。

### 3. 分頁與交互優化
- **指令鏈分頁修復**：修復了 `UIUtils` 中 `switchSmartTab` 的 `this` 綁定問題，確保「輔助說明」與「指令鏈」分頁切換功能恢復正常。
- **圖示與邊框細節**：縮小了收合圖示至 `14px`，並確保所有 header actions 正確靠右對齊。

### 4. 下一步行動
- **核心音訊優化**：持續監控並消除正弦波快速演奏時的極細微雜訊。
- **教學範例庫**：準備更多配合 FFT 頻譜分析儀的教學範例（如：濾鏡頻率響應實作）。
