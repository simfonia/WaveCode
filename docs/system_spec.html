1. 系統架構

WaveCode 採用異步排程（JS）與實時渲染（Rust）分離的架構，解決了傳統 Web Audio 易受主執行緒阻塞的痛點。

2. 核心技術實作

  • C 型樂器定義模型：對齊 #processing 創作邏輯，使用 C 型帽子積木封裝音源鏈。
    • 組件包含：振盪器、ADSR 包絡線、濾波器 (LP/HP)、音量控制。
    • 遞迴編譯：WaveCodeCompiler 遞迴掃描組件鏈並生成 Patch JSON 傳送至 Rust。
  • 硬同步 (Hard Sync)：每次觸發音符時，強制呼叫 unit.reset() 重置震盪器相位與濾波器狀態，徹底消除爆音與直流偏移。
  • 腳本安樂死機制 (Euthanasia)：
    • 透過 _execId 版本號控制。
    • 在所有循環 (Repeat/While/For) 中注入 isAlive(_id) 檢查。
    • 非同步 API (Sleep/PlayNote) 在恢復執行前檢查版本，不符則拋出 Script cancelled。
  • 批次鎖定 (Batch Locking)：在音訊回調的緩衝區層級進行 Mutex 鎖定，消除採樣點層級競爭，徹底解決機關槍雜訊。

3. 視覺化系統規格

  • Clipping 警報示波器：
    • 數據穿透：傳送未經過 clamp 的原始採樣數據至前端。
    • 越界繪圖：波形可繪製超出畫布邊界，模擬專業示波器過載現象。
    • CLIP 告警：偵測到振幅 > 1.0 時，波形變紅並顯示紅色 "CLIP" 字樣。
  • ADSR Glow 效果：
    • 發光光點：使用 SVG Gaussian Blur 濾鏡建立發光動態球體。
    • 動態 Sustain：光點在延持階段會呈正弦波來回擺動，增加生命感。

4. 模組化檔案規範

  • 積木定義：存放於 ui/src/blocks/，以功能模組命名 (如 audio_instruments.js)。
  • 產生器：存放於 ui/src/generators/javascript/，對應積木定義模組。
  • 命名空間：自定義積木統一使用 wc_ 前綴 (如 wc_text_print)。
  • i18n：所有標籤、提示與顏色數值 (Hues) 統一由 lang/ 語系檔定義。
